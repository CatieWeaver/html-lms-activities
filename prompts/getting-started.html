Prompt Guide

Two ways to approach a prompt

The prompts in this guide fall into two categories, and knowing which one fits your situation saves revision time.

Use a brainstorming prompt when you have content but aren't sure what interaction format would serve it best. Describe what you have, share your learning goals, and ask the AI to suggest a few formats before you commit to one. This often surfaces approaches worth considering that a more prescriptive prompt would bypass.

Use a prescriptive prompt when you already know the format you want. These prompts include the learning objective, the audience, and the specific technical parameters, and they produce more precise output because there is less for the AI to interpret. Both approaches are demonstrated below, and you can combine them: brainstorm first, then get prescriptive once you've chosen a direction.

01 Adapting an existing activity

The most reliable path to a finished activity is adapting one of the examples from the Activity Library. You can see exactly what you're getting, and the AI has a complete working model to build from rather than starting from scratch.

How to get the source code

Open any activity from the library. Right-click anywhere on the page and choose View Page Source (or press Ctrl+U on Windows / Cmd+Option+U on Mac). Select all the code (Ctrl+A), copy it (Ctrl+C), and paste it into your AI prompt.

Adaptation prompt template
I have an HTML [ACTIVITY TYPE] activity about [ORIGINAL SUBJECT]. I want to adapt it for my [YOUR SUBJECT] course to [YOUR LEARNING GOAL]. Keep all the functionality ([LIST KEY FEATURES]) but replace the content with my material below. Requirements: - Maintain WCAG 2.1 AA color contrast - Keep keyboard navigation and screen reader support - No external libraries or dependencies - Must work embedded in a Blackboard Ultra document My content: [PASTE YOUR CONTENT HERE] Here's the HTML to adapt: [PASTE SOURCE CODE]
Example
I have an HTML flashcard activity about psychology concepts. I want to adapt it for my Spanish 101 course to help students practice vocabulary before the Unit 2 exam. Keep the flip animation, study mode toggle, shuffle, and mastery tracking. Replace the psychology content with my vocabulary list. Requirements: - Maintain WCAG 2.1 AA color contrast - Keep keyboard navigation and screen reader support - No external libraries or dependencies - Must work embedded in a Blackboard Ultra document My vocabulary list: 1. la biblioteca — the library 2. el horario — the schedule [continue with remaining items] Here's the HTML to adapt: [paste source code]
02 Brainstorming prompt

Use this when you know your content but haven't settled on a format. Asking the AI to suggest options first — before building anything — often leads to a better activity than jumping straight to a specification.

Brainstorming prompt template
I'm working on [COURSE/MODULE/TOPIC] and have content that feels flat or hard for students to engage with. I want to make it more interactive. Here's the content or what I'm working with: [Paste your content, or describe it — for example: "I have a list of 12 leadership theories with brief descriptions" or "I need students to understand the stages of mitosis in sequence"] My learning goals for this content: - [e.g., Help students distinguish between closely related concepts] - [e.g., Give students a way to self-check before an assessment] Can you suggest 3 to 4 ways to present this as an interactive HTML activity that embeds in a Blackboard Ultra document? For each option, describe what it would look like and why it fits this content. All options should be self-contained HTML with no external libraries and should follow WCAG 2.1 AA standards.

Once you have options, follow up with the one that fits: "I like option 2. Can you build that using my content?" You can also mix and match: "Can you take the layout from option 1 and the interaction style from option 3?"

03 Activity-specific prompts

Use these when you know the format you want. Each prompt includes the key features to preserve and a clear structure for the content you need to supply.

Flashcards
Recall & Retrieval

Best for: Vocabulary, definitions, key concepts, formulas, terms. Any content where recognition and retrieval practice supports learning.

Create an HTML flashcard activity for [COURSE/TOPIC]. Create [NUMBER] flashcards where: - Front: [what goes on front — term, question, image description] - Back: [what goes on back — definition, answer, explanation] Required features: flip animation, study/quiz mode toggle, shuffle, mastery tracking (known / needs review), keyboard navigation (spacebar to flip, arrow keys to navigate), progress counter. My content: 1. [FRONT] — [BACK] 2. [FRONT] — [BACK] Accessibility: WCAG 2.1 AA contrast, full keyboard support, ARIA labels, screen reader compatible. Single HTML file, no external libraries, Blackboard Ultra compatible.
Sorting & Categorization
Classification

Best for: Any content requiring classification — literary devices, historical causes, business models, clinical presentations, data types.

Create an HTML drag-and-drop sorting activity for [COURSE]. Students will sort [NUMBER] items into [NUMBER] categories. Categories: 1. [CATEGORY NAME]: [brief description] 2. [CATEGORY NAME]: [brief description] Items to sort (with correct placement): - [ITEM] — belongs in [CATEGORY] - [ITEM] — belongs in [CATEGORY] Required features: drag-and-drop, immediate feedback with explanation on placement, progress tracking, reset button. Include a keyboard alternative (Tab to select, Enter to pick up and drop) for accessibility. WCAG 2.1 AA contrast, ARIA live regions for feedback. Single HTML file, no external libraries.
Branching Scenarios & Decision Trees
Critical Thinking

Best for: Case studies, clinical reasoning, ethical dilemmas, troubleshooting. Scenarios where the right answer depends on context and teaches through consequence rather than instruction.

Create an HTML branching scenario for my [COURSE/CONTEXT]. Starting scenario: [Describe the initial situation — who, what, where, and what decision needs to be made] Decision points: 1. [SITUATION] Options: A) [CHOICE] B) [CHOICE] C) [CHOICE] A leads to: [outcome or next scenario] B leads to: [outcome or next scenario] 2. [Continue as needed] Each outcome should explain why the choice led there and what the learner should take away. Include a restart option. Required features: one decision point visible at a time, progress indicator, outcome explanations, restart button. Keyboard navigable, screen reader compatible, WCAG 2.1 AA. Single HTML file.
Checklists & Procedure Guides
Procedural Knowledge

Best for: Multi-step procedures, safety protocols, pre-activity preparation, project milestones. Content where sequence and completion both matter.

Create an HTML checklist for [PROCEDURE/PROTOCOL] in my [COURSE] course. Sections: 1. [SECTION NAME] - [STEP/ITEM] - [STEP/ITEM] 2. [SECTION NAME] - [STEP/ITEM] Required features: section-level and overall progress tracking, checkboxes that persist during the session, print-friendly output, clear visual hierarchy between sections and items. Accessibility: large click targets, visible focus states, keyboard navigable, WCAG 2.1 AA contrast. Single HTML file, no external libraries, Blackboard Ultra compatible.
Timelines & Sequencing
Temporal Reasoning

Best for: Chronological events, process sequences, historical periods, scientific discovery timelines, stages in a model or framework.

Create an HTML timeline sequencing activity for [TOPIC]. Students will arrange events in chronological order. Events: 1. [DATE/PERIOD]: [EVENT NAME] — [brief description] Significance: [why this matters] 2. [continue...] Required features: click-to-place interaction, immediate feedback when an event is placed correctly or incorrectly, contextual detail revealed after correct placement, progress tracking, reset option. Accessibility: keyboard navigation, clear visual feedback, ARIA announcements for correct and incorrect placement, WCAG 2.1 AA. Single HTML file, no external libraries.
Error Identification
Analytical Thinking

Best for: Editing and revision practice, code debugging, identifying flawed reasoning, spotting logical fallacies, APA or citation error correction.

Create an HTML error identification activity for my [COURSE]. A passage with [NUMBER] embedded errors, where students click highlighted text to reveal what's wrong and how to fix it. Topic/context: [describe the passage] Error types to include: - [ERROR TYPE 1 — e.g., subject-verb disagreement] - [ERROR TYPE 2] - [ERROR TYPE 3] For each error, provide: - What's wrong - The corrected version - The underlying rule or principle Required features: click-to-reveal feedback, progress counter, summary at completion showing all errors and corrections. Accessibility: errors indicated by both color and underline (not color alone), keyboard navigable, screen reader announces feedback, WCAG 2.1 AA. Single HTML file, no external libraries.
04 Tips for better results

Specificity is the single biggest lever. Compare these two prompts for the same task:

Make an activity for an introductory biology course.
Create flashcards for 15 biology terms related to cellular respiration for an undergraduate intro biology course. Include ATP, mitochondria, glycolysis, and the electron transport chain. Each card should have the term on the front and a 1 to 2 sentence definition with one real-world example on the back.
Provide your own content

Supplying your own content means the activity reflects your course's specific examples, terminology, and framing. AI can generate plausible content, but only you know what fits your students and learning objectives.

Include accessibility requirements

Specify WCAG 2.1 AA contrast, keyboard navigation, ARIA labels, and screen reader compatibility in every prompt. These need to be requirements from the start, not requests in a revision pass.

Test in your actual LMS

Browser testing is a good first check, but LMS environments handle HTML differently. Always paste code into your LMS editor and test every interactive element before sharing with students.

Describe what you're seeing

When revising, tell the AI what is happening versus what you expected: "The cards aren't flipping when I click them" gets better results than "fix the flashcards." Specific observations lead to targeted fixes.

05 Troubleshooting in your LMS

LMS environments process HTML differently than browsers do. Most issues follow a pattern, and each one has a targeted prompt you can use to address it.

Excess white space or unexpected gaps
Why it happens: Your LMS adds its own spacing around the content block, which stacks on top of the activity's own margins and padding.

Fix prompt: "When I paste this into my LMS, there's too much white space above and below the content. Reduce all top and bottom padding and margins by about half, and remove any unnecessary line breaks."
Activity scrolls inside a small box instead of expanding
Why it happens: Fixed height values or overflow settings are constraining the container.

Fix prompt: "The activity is displaying inside a scrollable box in my LMS instead of expanding to full height. Remove all fixed height values and change any overflow: scroll or overflow: auto to overflow: visible."
Icons not displaying
Why it happens: Most LMS editors strip external stylesheet references, so icon libraries that load from an external source won't work.

Fix prompt: "The icons in this activity aren't displaying in my LMS. Replace all icon library references with inline SVGs or accessible Unicode characters that don't rely on external files."
Buttons not responding or activity not advancing
Why it happens: JavaScript that runs before the HTML has fully loaded, or a mismatch between element IDs in the HTML and the script.

Fix prompt: "The buttons in this activity aren't working when I test it in my LMS. Check that all JavaScript is wrapped in a DOMContentLoaded event listener, and verify that every ID referenced in the script exactly matches an ID in the HTML."
Elements overlapping or layout breaking
Why it happens: Absolute positioning or fixed widths that work in isolation but collapse inside the LMS content container.

Fix prompt: "Parts of this activity are overlapping or misaligned in my LMS. Remove any absolute or fixed positioning, convert fixed pixel widths to percentages or max-width values, and simplify to a single-column layout if needed."
General approach: Whether you address issues one at a time or in a single pass, what matters most is giving the AI a precise description of what you're seeing and where. "The drag-and-drop items aren't snapping into the drop zones" will get a more useful response than "it doesn't work." The more detail you provide about the specific problem, the less guesswork the AI has to do.
